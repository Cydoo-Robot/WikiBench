## 2. 项目定位

### 2.1 一句话定位

> **WikiBench 是首个面向 LLM 维护型知识库（LLM Wiki）的系统性评测基础设施。**

### 2.2 三个核心判断

1. **LLM Wiki 是真·新范式**，不是 RAG 的换皮
   - RAG：每次查询都去 embedding + retrieve
   - LLM Wiki：LLM 一次性"编译"原始语料为结构化知识，后续直接查询编译产物
   - 改变了 *token 经济学*（预付 ingest 成本换持续查询节省）

2. **评测标准的空白是结构性的，不是暂时的**
   - 质量、效率、可维护性三个维度**相互制衡**（权衡型指标）
   - 单一指标会诱导错误优化方向（只压 token 会丢失保真度）
   - 没有公共数据集 + 公共指标 → 无法跨项目对比

3. **基础设施级机会窗口已开启，但会快速关闭**
   - 类比：SWE-bench 2023 年 10 月发布，6 个月内成为代码 Agent 事实标准
   - 生态内第一个被广泛采用的标准会形成*命名权*与*默认值效应*
   - 估计窗口期 **3–6 个月**

### 2.3 我们不是什么

| ❌ 不是 | ✅ 是 |
|---|---|
| 又一个 LLM Wiki 实现 | 评测所有 LLM Wiki 实现的**度量衡** |
| RAG benchmark | 专门针对 *知识编译* 范式 |
| 纯效率跑分工具 | 质量 + 效率 + 可维护性三维评测 |
| 闭源商业产品 | Apache 2.0 开源，公开 leaderboard |

## 3. 与现有基准的差异

| 基准 | 评测对象 | 核心维度 | 与 WikiBench 的关系 |
|------|---------|---------|-------------------|
| **BEIR** | 信息检索系统 | 检索精度 | 借鉴：多数据集标准化评估范式 |
| **LongMemEval** | 长程对话记忆 | 多轮召回 | 借鉴：长时间窗口下的信息保真度 |
| **FRAMES** | 多文档推理 | 跨文档事实推理 | 借鉴：跨源事实一致性 |
| **SWE-bench** | 代码 Agent | 真实世界任务可验证性 | 借鉴：synthetic+real 混合数据集范式 |
| **RAGAS** | RAG 系统 | 检索+生成质量 | **区别**：RAGAS 测"查询时"，WikiBench 测"编译后" |

**WikiBench 的独特性**：首个把 *知识编译产物本身* 作为评测对象，而不是把它当成中间产物去测下游生成。

## 4. 分阶段形态

本项目规划两个形态，**当前聚焦 Phase 1**：

### Phase 1 · WikiBench（社区评测套件）

- **目标**：成为 LLM Wiki 生态的公共评测标准
- **交付**：标准语料库 + 统一评测接口 + 公开排行榜
- **受众**：开源作者、研究员、技术决策者
- **时间窗**：0–6 个月

### Phase 2 · WikiBench（企业级诊断框架）

- **目标**：让企业团队能持续监控自己知识库的生产健康度
- **交付**：
  - `probe` 子命令：对企业自有语料做一次性健康检查
  - `monitor` 子命令：集成 CI/CD，每次 wiki 更新后自动评测
  - 长期趋势报告与告警
- **受众**：企业技术团队、平台团队、SRE
- **时间窗**：6–18 个月

> **为什么分两阶段？**
> WikiBench（社区跑分）先建立标准和品牌认知 → WikiBench（企业诊断）基于同一套核心自然演进。避免一开始就做企业版导致标准失去"中立公共"属性。

## 5. 核心价值主张

### 对开源作者
- 有了可被引用的质量指标（而不是只报 token 节省）
- 有标准语料跑 regression test，防止 refactor 掉质量

### 对研究员
- 有可复现的数据集和任务
- 有横向对比不同实现的统一坐标系

### 对企业技术决策者
- 在上生产前**先知道在自己数据规模上质量如何**
- 换底层模型、换实现、数据增长后，**知识腐烂到什么程度**

### 对整个生态
- 把"哪个实现更好"从主观讨论变成**可测量的问题**
- 自然推动整个生态向质量靠拢，而不是军备竞赛式压 token

---

## 6. 成功标准（3 个里程碑）

| 阶段 | 标志 | 量化指标 |
|------|------|---------|
| **立足** | MVP 跑通 | synthetic corpus 生成器 + 3 类任务 + 1 个参考实现接入 |
| **被采用** | 成为事实标准 | GitHub ≥ 1k star；≥ 5 个主流 LLM Wiki 实现官方接入；排行榜周活 |
| **基础设施化** | 被其他工具依赖 | 被至少 2 个第三方工具（如 LangGraph、LlamaIndex）引用 / 集成 |
