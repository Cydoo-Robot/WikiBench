# WikiBench 项目规划文档

> 本目录是 WikiBench 项目的完整规划文档，基于根目录 `README_CN.md` 中的初始构想，经过深度拆解和工程化设计而成。
>
> **项目一句话定位**：*首个面向 LLM 维护型知识库（LLM Wiki）的系统性评测基础设施* —— 对标 SWE-bench 之于代码 Agent、BEIR 之于信息检索。

---

## 背景速览

Karpathy 提出 **LLM Wiki** 概念后两周内出现了 15+ 开源实现（ussumant、Ar9av、AgriciDaniel、claude-obsidian 等）。核心范式：让 LLM 把原始文档"编译"成结构化知识库，取代 RAG"每次查询都检索"的无状态模式。

**当前生态缺口**：所有实现的"测试"本质上都是作者跑一遍自己的项目、报告 token 节省比例——**只有效率、没有质量；只有一个数据点、没有控制变量；没有跨项目可复现性**。

**窗口期判断**：谁先建立起质量评测标准，谁就自然成为这一生态的基础设施。

## 阅读建议

- **5 分钟速览**：`00` 的"一张图看懂"+ `07` 的 MVP 范围。
- **开始动手前**：00 → 02 → 05 → 06 → 07（按顺序读完再开工）。
- **遇到指标问题**：→ 04。
- **遇到数据集/规模问题**：→ 03。
- **遇到爬虫/论坛数据问题**：→ 09。
- **遇到外部接入/leaderboard 问题**：→ 08。

## 文档约定

- 所有文档使用简体中文为主，关键术语保留英文（如 *corpus*、*fidelity*、*staleness*）。
- 每份文档顶部都有"**本文档回答什么问题**"，末尾有"**未决议题**"清单。
- 代码示例统一使用 Python 3.11+ 语法。
- 目录结构、接口签名等"规范性"内容在 `05`、`06` 两份文档中保持唯一源（single source of truth）。
