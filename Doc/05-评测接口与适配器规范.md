## 2. 核心数据模型

### 2.1 `Document`

> **核心约定：corpus 里的所有文档，无论来源如何，最终都以 Markdown（`.md`）格式存储和传递给 adapter。**
>
> `modality` 字段描述的是文档的**语义来源**，不是文件格式：
> - 论坛帖子（`forum_thread`）爬取后由 `ForumRenderer` 渲染成 md 再写入 corpus
> - 会议记录（`transcript`）、邮件（`email`）等同理，都先转成 md 再打包
> - adapter 收到的永远是 `content: str`（Markdown 文本）+ `modality`（来源标签）
>
> 这样设计的好处：adapter 不需要解析多种格式，所有 LLM Wiki 实现天然就处理 md；`modality` 仅供 adapter 选择性地调整提示策略。

```python
from pydantic import BaseModel, Field
from datetime import datetime
from typing import Literal

Modality = Literal["markdown", "transcript", "chat", "email", "code_comment", "forum_thread", "other"]

class Document(BaseModel):
    id: str
    path: str                              # 虚拟路径，保留目录结构语义（始终以 .md 结尾）
    content: str                           # Markdown 文本，所有来源统一格式
    modality: Modality = "markdown"
    timestamp: datetime | None = None      # 原始创建时间（用于 staleness）
    metadata: dict[str, str] = Field(default_factory=dict)
```

### 2.2 `Query` / `QueryResponse`

```python
class Query(BaseModel):
    id: str
    text: str
    intent: Literal["qa", "fidelity_check", "contradiction_check", "grounding"]
    params: dict[str, object] = Field(default_factory=dict)
    # 例：fidelity_check 时 params={"claim": "...", "decision_space": ["supported", "not_supported", "unknown"]}

class QueryResponse(BaseModel):
    answer: str                            # 主回答，自由文本
    structured: dict[str, object] = Field(default_factory=dict)
    # 不同 intent 要求不同 structured 字段：
    #   qa               → {}（可选 "candidates": [...]）
    #   fidelity_check   → {"verdict": "supported|not_supported|unknown"}
    #   contradiction    → {"has_contradiction": bool, "pairs": [...]}
    #   grounding        → {"sources": ["doc1#L10", ...]}
    sources: list[str] = Field(default_factory=list)
    confidence: float | None = None
    raw: dict[str, object] | None = None   # adapter 保留原始返回供调试
```

### 2.3 `IngestResult` / `RunStats`

```python
class IngestResult(BaseModel):
    wiki_id: str                           # adapter 内部的 wiki 标识
    stats: IngestStats

class IngestStats(BaseModel):
    tokens_in: int
    tokens_out: int
    llm_calls: int
    duration_s: float
    wiki_tokens: int                       # 编译产物 token 数，必须报告
    cost_usd: float | None = None
    extra: dict[str, object] = Field(default_factory=dict)
```

### 2.4 `BenchmarkResult`

```python
class BenchmarkResult(BaseModel):
    run_id: str                            # uuid
    created_at: datetime
    impl: str                              # e.g. "ussumant@0.3.2"
    corpus_id: str                         # e.g. "synthetic-001-saas@1.0.0"
    environment: RunEnvironment
    ingest: IngestStats
    per_task: dict[str, TaskResult]
    metrics: dict[str, float]              # 扁平化指标字典
    composite: dict[str, float]            # 多套 composite 分数
    warnings: list[str] = Field(default_factory=list)

class RunEnvironment(BaseModel):
    python_version: str
    os: str
    wikibench_version: str
    impl_version: str
    impl_commit: str | None = None
    llm_models: dict[str, str]             # {"adapter_backend": "gpt-4o-mini", "judge": "..."}
    seed: int
```

## 3. `WikiAdapter` 抽象基类

### 3.1 最小实现（MVP 必需方法）

```python
from abc import ABC, abstractmethod

class WikiAdapter(ABC):
    """第三方 LLM Wiki 实现接入点。"""

    # ——— 元数据（类属性）———
    name: str                    # adapter 唯一 ID，例如 "ussumant"
    version: str                 # 版本号
    supports_update: bool = False
    supports_staleness: bool = False
    supports_coverage: bool = False

    # ——— 生命周期 ———
    def __init__(self, config: dict):
        """从配置字典初始化。config schema 由 adapter 自行声明。"""

    def describe(self) -> dict:
        """返回 adapter 元数据，用于写进 RunEnvironment。必须包含 llm_models。"""

    # ——— 必需方法 ———
    @abstractmethod
    def ingest(self, docs: list[Document]) -> IngestResult:
        """把原始文档编译成 wiki，返回编译统计。"""

    @abstractmethod
    def query(self, query: Query) -> QueryResponse:
        """对已编译 wiki 提问。必须填充 structured 字段以满足 intent。"""

    # ——— 可选方法 ———
    def update(
        self,
        added: list[Document],
        removed: list[str],
        modified: list[Document],
    ) -> IngestResult:
        raise NotImplementedError

    def export_wiki(self) -> str:
        """可选：导出编译产物，用于度量 wiki_tokens。不支持时 runner 会用采样估计。"""
        raise NotImplementedError

    def teardown(self) -> None:
        """释放资源（关闭进程、清理缓存等）。"""
        pass
```

### 3.2 最小可行 Adapter 示例（教学用）

```python
# wikibench/adapters/builtin/naive.py
import tiktoken
from wikibench.adapters import WikiAdapter
from wikibench.models import Document, Query, QueryResponse, IngestResult, IngestStats

class NaiveAdapter(WikiAdapter):
    """基线：把所有文档拼成一个大 prompt，每次查询都塞全量上下文。"""
    name = "naive"
    version = "0.1.0"

    def __init__(self, config):
        self.model = config.get("model", "gpt-4o-mini")
        self.llm = make_llm_client(self.model)
        self._context: str = ""

    def describe(self):
        return {"llm_models": {"adapter_backend": self.model}}

    def ingest(self, docs):
        self._context = "\n\n".join(f"# {d.path}\n{d.content}" for d in docs)
        enc = tiktoken.encoding_for_model(self.model)
        wiki_tokens = len(enc.encode(self._context))
        return IngestResult(
            wiki_id="naive-0",
            stats=IngestStats(
                tokens_in=0, tokens_out=0, llm_calls=0,
                duration_s=0.01, wiki_tokens=wiki_tokens,
            ),
        )

    def query(self, query):
        prompt = self._build_prompt(query, self._context)
        reply = self.llm.complete(prompt)
        return self._parse(query, reply)
```

## 4. Task 接口

```python
class Task(ABC):
    id: str
    version: str

    @abstractmethod
    def prepare(self, corpus: Corpus) -> list[Query]: ...

    @abstractmethod
    def score(self, query: Query, response: QueryResponse, corpus: Corpus) -> Score: ...

    def aggregate(self, scores: list[Score]) -> TaskResult:
        """默认实现：mean + 分层切片。子类可覆盖。"""
```

**注册机制**：

```python
# wikibench/tasks/__init__.py
from wikibench.registry import register_task

@register_task

    id = "retrieval_accuracy"
    version = "1.0.0"
    ...
```

## 5. Metric 注册

```python
from wikibench.registry import register_metric

@register_metric(id="hallucination_rate", higher_is_better=False, range=(0.0, 1.0))
def hallucination_rate(task_results: dict[str, TaskResult]) -> float:
    fidelity = task_results["knowledge_fidelity"]
    return fidelity.slices["negative_samples"]["false_positive_rate"]
```

## 6. Runner 调用契约

```python
from wikibench import Runner

result = Runner(
    adapter_spec="ussumant:0.3.2",          # 或传 Adapter 实例
    adapter_config={"model": "gpt-4o-mini"},
    corpus="synthetic-001-saas@1.0.0",      # 或 Corpus 实例
    tasks=["retrieval_accuracy", "knowledge_fidelity", "contradiction_detection"],
    seed=42,
    cache_dir="./.wikibench-cache",
).run()

result.save("./runs/ussumant-20260417/")
```

**Runner 的保证**：
- 所有 LLM 调用经过 `TokenCounter`，token 统计与 adapter 自报对账
- 每个 query 经过 `ResponseCache`（基于 corpus_version + adapter_version + query hash）
- 超时保护：单 query 默认 60s，ingest 默认 30min，可配置
- 故障时 partial result 仍可导出

## 7. 成本追踪中间件

所有 LLM 调用（adapter 内部 + judge）都必须经过统一入口，否则计费失真：

```python
from wikibench.runtime import llm_call

# adapter 内部
reply = llm_call(
    model="gpt-4o-mini",
    messages=[...],
    purpose="adapter.ingest",     # 分类，便于分摊
)
```

- `llm_call` 自动记录 tokens_in/out、latency、cost
- 按 `purpose` 分桶：`adapter.ingest` / `adapter.query` / `task.judge` / `corpus.generate`
- 报告中分开列出各类成本

**如果 adapter 不想用内置入口**：必须在 `describe()` 里声明 `self_reported_tokens=True`，并在每次 `query()` 返回中正确填充 `IngestStats`/token 字段。Runner 会做一致性抽样校验。

## 8. 缓存策略

### 8.1 Ingest 缓存

Key = `hash(corpus_version, adapter_name, adapter_version, adapter_config, seed)`
- 命中时跳过 ingest，复用上一次 wiki 产物
- 默认开启，`--no-cache` 关闭

### 8.2 Query 缓存

Key = `hash(wiki_id, query.text, query.intent, query.params, adapter_version)`
- 跑同一组 task 多次时大幅省成本
- adapter 可声明 `deterministic=False` 禁用

## 9. 插件发现（entry_points）

第三方 adapter 以独立 pip 包分发，通过 entry_point 注册：

```toml
# pyproject.toml of wikibench-adapter-ussumant
[project.entry-points."wikibench.adapters"]
ussumant = "wikibench_adapter_ussumant:UssumantAdapter"
```

用户侧：

```bash
pip install wikibench-adapter-ussumant
wikibench run --impl ussumant --corpus small
```

同理支持 `wikibench.tasks`、`wikibench.metrics`、`wikibench.corpora`、`wikibench.reporters`。

## 10. Adapter 接入 checklist（给贡献者）

- [ ] 继承 `WikiAdapter`，实现 `ingest` 和 `query`
- [ ] 声明 `name`、`version`、能力标志
- [ ] 在 `describe()` 返回所用 LLM 模型列表
- [ ] 至少支持 `intent=qa` 和 `intent=fidelity_check`
- [ ] 在 README 说明最小 config schema 和硬件/ API 依赖
- [ ] 通过 `wikibench verify-adapter <name>` 自检（框架提供 smoke test）
- [ ] 在 `examples/` 下附一个最小可跑 demo
- [ ] 提交 PR 把 adapter 信息加入官方 `adapters.yaml` 索引

**`wikibench verify-adapter`** 自动跑：
1. 超小 synthetic corpus（5 docs）ingest
2. 5 个 QA / 5 个 claim / 2 个矛盾
3. 校验返回 schema、token 报账一致性
4. 产出通过/失败报告

## 11. 版本兼容策略

- Adapter 接口遵循 SemVer
- `WikiAdapter` 类添加新**可选**方法 → MINOR
- 修改必需方法签名 → MAJOR，并提供迁移指南
- 框架在 runtime 校验 adapter 声明的 `wikibench>=X.Y`，不兼容时拒绝运行
